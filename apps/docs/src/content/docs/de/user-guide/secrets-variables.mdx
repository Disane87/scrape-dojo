Passwörter, API-Keys, Login-Daten -- so etwas gehört nicht in Klartext in deine Scrape-Konfiguration. Scrape Dojo hat ein durchdachtes System für verschlüsselte Secrets und flexible Variablen eingebaut. Lass uns mal anschauen, wie das funktioniert.

Kurz gesagt: Secrets sind für sensible Daten, Variablen für alles andere. Beides kannst du bequem in deinen Scrapes per Template-Syntax verwenden.

## 🔒 Secrets

Secrets werden mit **AES-256-CBC** verschlüsselt in der Datenbank gespeichert. Sie sind ideal für:

- Passwörter und Login-Daten
- API-Keys und Tokens
- OAuth Client Secrets
- Sonstige vertrauliche Informationen

### 🆕 Secrets erstellen

#### Über die UI

1. Einstellungen öffnen (Zahnrad-Icon)
2. Zum Tab "Secrets" navigieren
3. "Neues Secret" anklicken
4. Name, Wert und optionale Beschreibung eingeben
5. Speichern -- der Wert wird automatisch verschlüsselt

#### Über die API

```bash
POST /api/secrets
Content-Type: application/json

{
  "name": "portalPassword",
  "value": "geheimes-passwort",
  "description": "Login-Passwort für das Portal"
}
```

<Aside type="note">
Der Wert wird in der API-Response **niemals** zurückgegeben. Du siehst nur Metadaten wie Name, Beschreibung und Erstellungsdatum.
</Aside>

#### Über Umgebungsvariablen

```bash
SCRAPE_DOJO_SECRET_PORTAL_PASSWORD=geheimes-passwort
SCRAPE_DOJO_SECRET_API_TOKEN=abc123xyz
```

Beim Start synchronisiert Scrape Dojo diese automatisch in die Datenbank. Die Benennung wird in camelCase konvertiert:
- `SCRAPE_DOJO_SECRET_PORTAL_PASSWORD` wird zu `portalPassword`

### 🛠️ Secrets verwalten (CRUD)

```bash
# Alle Secrets auflisten (ohne Werte)
GET /api/secrets

# Einzelnes Secret abrufen (ohne Wert)
GET /api/secrets/{id}

# Secret aktualisieren
PUT /api/secrets/{id}
{ "value": "neuer-wert", "description": "Aktualisierte Beschreibung" }

# Secret löschen
DELETE /api/secrets/{id}
```

### 🔗 Secrets mit Workflows verknüpfen

Secrets können mit bestimmten Workflows verknüpft werden, um den Zugriff einzuschränken. Bei der Erstellung kannst du eine Workflow-ID angeben.

### 📝 Secrets in Konfigurationen verwenden

In deinen Scrape-Konfigurationen referenzierst du Secrets einfach per Template-Syntax:

```jsonc
{
  "action": "type",
  "name": "Passwort eingeben",
  "params": {
    "selector": "input[type='password']",
    "text": "{{secrets.portalPassword}}"
  }
}
```

<Aside type="caution">
Secrets werden erst zur Laufzeit aufgelöst. In der Konfigurationsdatei steht nur der Platzhalter -- der eigentliche Wert verlässt nie die verschlüsselte Datenbank, bis er gebraucht wird.
</Aside>

## 📦 Workflow-Variablen

Variablen werden im `metadata.variables`-Array einer Scrape-Konfiguration definiert. Wenn du einen Scrape über die UI startest, zeigt dir Scrape Dojo automatisch ein Eingabeformular für die definierten Variablen an.

### 📋 Variablen-Typen

| Typ | Beschreibung | UI-Element |
|-----|--------------|------------|
| `string` | Freitext-Eingabe | Textfeld |
| `password` | Verdeckte Eingabe | Passwortfeld |
| `number` | Numerischer Wert | Zahlenfeld |
| `boolean` | Wahr/Falsch | Checkbox |
| `email` | E-Mail-Adresse | E-Mail-Feld |
| `url` | URL-Eingabe | URL-Feld |
| `select` | Auswahl aus Optionen | Dropdown |

### 📐 Variable-Definition

```jsonc
{
  "metadata": {
    "variables": [
      {
        "name": "targetYear",
        "label": "Zieljahr",
        "type": "number",
        "default": 2025,
        "required": true,
        "description": "Welches Jahr soll verarbeitet werden?"
      },
      {
        "name": "region",
        "label": "Region",
        "type": "select",
        "default": "de",
        "options": [
          { "value": "de", "label": "Deutschland" },
          { "value": "at", "label": "Österreich" },
          { "value": "ch", "label": "Schweiz" }
        ]
      },
      {
        "name": "maxPages",
        "label": "Maximale Seiten",
        "type": "number",
        "default": 10,
        "placeholder": "z.B. 10"
      },
      {
        "name": "email",
        "label": "E-Mail",
        "type": "email",
        "secretRef": "portalEmail",
        "description": "Wird aus Secret geladen"
      }
    ]
  }
}
```

<Aside type="tip">
Variablen mit `secretRef` werden automatisch mit dem Wert des referenzierten Secrets befüllt. Der Benutzer kann den Wert trotzdem im UI-Dialog überschreiben.
</Aside>

### 🔧 Variablen in Templates verwenden

```jsonc
{
  "action": "navigate",
  "name": "Seite öffnen",
  "params": {
    "url": "https://example.com/data?year={{variables.targetYear}}&region={{variables.region}}"
  }
}
```

## 🚀 Runtime-Variablen

Runtime-Variablen werden beim Start eines Scrapes übergeben -- entweder über den UI-Dialog oder den API-Body:

```bash
POST /api/scrape/my-workflow
Content-Type: application/json

{
  "variables": {
    "targetYear": 2023,
    "maxPages": 5
  }
}
```

## 💾 Datenbank-Variablen

Globale oder Workflow-spezifische Variablen, die über die API gespeichert werden:

```bash
# Variable erstellen
POST /api/variables
{
  "name": "defaultYear",
  "value": "2025",
  "scope": "global",
  "description": "Standard-Jahr für alle Scrapes"
}

# Alle Variablen abrufen
GET /api/variables

# Globale Variablen
GET /api/variables/global

# Workflow-spezifische Variablen
GET /api/variables/workflow/{workflowId}

# Variable aktualisieren
PUT /api/variables/{id}
{ "value": "2026" }

# Variable löschen
DELETE /api/variables/{id}
```

Datenbank-Variablen können auch über Umgebungsvariablen mit dem Prefix `SCRAPE_DOJO_VAR_` definiert werden:

```bash
SCRAPE_DOJO_VAR_DEFAULT_YEAR=2025
SCRAPE_DOJO_VAR_API_ENDPOINT=https://api.example.com
```

## 📊 Auflösungs-Priorität

Wenn eine Variable an mehreren Stellen definiert ist, gilt diese Reihenfolge (höchste zuerst):

1. **Runtime** -- Werte, die beim Start via UI-Dialog oder API-Body übergeben werden
2. **Database** -- Über `/api/variables` oder `SCRAPE_DOJO_VAR_*` gespeicherte Werte
3. **Config Defaults** -- In `metadata.variables[].default` definierte Standardwerte

### 💡 Beispiel

```bash
# Umgebungsvariable
SCRAPE_DOJO_VAR_TARGET_YEAR=2024
```

```jsonc
// Workflow-Konfiguration
{
  "metadata": {
    "variables": [
      { "name": "targetYear", "type": "number", "default": 2025 }
    ]
  }
}
```

```bash
# API-Aufruf mit Runtime-Variable
POST /api/scrape/my-workflow
{ "variables": { "targetYear": 2023 } }
```

Ergebnis: `{{variables.targetYear}}` ergibt **2023** (Runtime gewinnt).

Ohne Runtime-Variable wäre der Wert **2024** (Database), und ohne Database-Variable wäre er **2025** (Config Default).

## 💾 storeData mit persist

Die `storeData`-Action kann Daten zwischen Runs persistieren:

```jsonc
{
  "name": "save-results",
  "action": "storeData",
  "params": {
    "key": "myWorkflow.lastResults",
    "value": "{{previousData.transformedData}}",
    "persist": true
  }
}
```

Mit `persist: true` bleiben die Daten über mehrere Ausführungen hinweg erhalten. Gespeicherte Daten sind über `{{storedData.myWorkflow.lastResults}}` in nachfolgenden Runs zugänglich.

## 🔐 Verschlüsselung

### 🔑 Algorithmus

Secrets werden mit **AES-256-CBC** verschlüsselt:
- Schlüssellänge: 256 Bit (32 Bytes)
- Der Verschlüsselungskey wird über `SCRAPE_DOJO_ENCRYPTION_KEY` konfiguriert
- Jedes Secret erhält einen eigenen Initialisierungsvektor (IV)

### 🔧 Key generieren

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

Der generierte 64-Zeichen-Hex-String wird als `SCRAPE_DOJO_ENCRYPTION_KEY` in der `.env`-Datei hinterlegt.

<Aside type="caution">
**Wichtige Hinweise:**
- Den Encryption Key **niemals** in ein Repository committen
- Den Key sicher in einem Passwort-Manager aufbewahren
- Bei Verlust des Keys sind alle gespeicherten Secrets **unwiderruflich verloren**
- Für Key-Rotation müssen alle Secrets nach dem Key-Wechsel neu erstellt werden
</Aside>
