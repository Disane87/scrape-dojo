Manchmal willst du eine Loop vorzeitig beenden — z.B. wenn du das gefunden hast, was du suchst, oder wenn ein bestimmter Schwellenwert erreicht ist. Genau dafür gibt's `break`. Du gibst eine JSONata-Bedingung an, und wenn die `true` ergibt, wird die aktuelle Loop sofort abgebrochen.

## Parameter

| Parameter | Typ | Pflicht | Beschreibung |
|-----------|-----|---------|--------------|
| `condition` | `string` | Ja | JSONata-Ausdruck, der gegen `previousData`, `currentData` und `storedData` evaluiert wird |

<Aside type="note">
Die `condition` wird als JSONata-Ausdruck ausgewertet — nicht als Handlebars-Template. Dir stehen dabei `previousData`, `currentData` und `storedData` als Kontextobjekte zur Verfügung.
</Aside>

## Beispiele

**Abbruch wenn Preis über 100:**

```jsonc
{
  "name": "priceLoop",
  "action": "loop",
  "params": {
    "elementKey": "products",
    "actions": [
      {
        "name": "getPrice",
        "action": "extract",
        "params": {
          "selector": ".price",
          "extractData": "innerText",
          "transformData": "$number($replace($, ',', '.'))"
        }
      },
      {
        "name": "stopIfExpensive",
        "action": "break",
        "params": {
          "condition": "previousData.getPrice > 100"
        }
      }
    ]
  }
}
```

**Abbruch bei leerem Ergebnis:**

```jsonc
{
  "name": "pageLoop",
  "action": "loop",
  "params": {
    "elementKey": "pageNumbers",
    "actions": [
      {
        "name": "loadPage",
        "action": "navigate",
        "params": { "url": "https://example.com/results?page={{currentData.pageLoop.value}}" }
      },
      {
        "name": "results",
        "action": "extract",
        "params": {
          "selector": ".result-item",
          "extractData": "innerText",
          "multiple": true
        }
      },
      {
        "name": "stopIfEmpty",
        "action": "break",
        "params": {
          "condition": "$count(previousData.results) = 0"
        }
      }
    ]
  }
}
```

<Aside type="tip">
`break` funktioniert nur innerhalb einer `loop`-Action. Wenn die Bedingung `false` ergibt, passiert einfach nichts und die Loop läuft normal weiter.
</Aside>
