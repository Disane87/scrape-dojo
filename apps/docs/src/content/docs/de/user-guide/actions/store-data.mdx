Mit `storeData` kannst du Werte abspeichern, die du später im selben Run oder sogar in zukünftigen Runs wieder verwenden willst. Das ist super praktisch für Zähler, Zwischenergebnisse oder Flags, die du über mehrere Steps hinweg brauchst.

## ⚙️ Parameter

| Parameter | Typ | Pflicht | Beschreibung |
|-----------|-----|---------|--------------|
| `key` | `string` | Ja | Schlüssel, unter dem der Wert gespeichert wird |
| `value` | `any` | Ja | Der zu speichernde Wert (unterstützt Handlebars-Templates) |
| `persist` | `boolean` | Nein | Wenn `true`, wird der Wert in der Datenbank persistiert und überlebt alle Runs (Job-Level) |
| `attachToRun` | `boolean` | Nein | Wenn `true`, wird der Wert dem aktuellen Run zugeordnet und in den Run-Details angezeigt (Standard: `true`) |

### 📊 Speicher-Ebenen

- **Runtime** (Standard): Der Wert ist im aktuellen Run über `storedData.key` verfügbar
- **Run-Level** (`attachToRun: true`): Der Wert wird in der Datenbank gespeichert und ist in der Run-Detail-Ansicht sichtbar
- **Job-Level** (`persist: true`): Der Wert bleibt über alle Runs hinweg erhalten — perfekt für inkrementelles Scraping

<Aside type="note">
Wenn `persist: true` gesetzt ist, wird `attachToRun` automatisch auf `false` gesetzt. Job-Level-Daten gehören zum Scrape, nicht zu einem einzelnen Run.
</Aside>

## 💡 Beispiele

**Einfacher Wert speichern:**

```jsonc
{
  "name": "saveTotal",
  "action": "storeData",
  "params": {
    "key": "totalProducts",
    "value": "{{previousData.productCount}}"
  }
}
```

Danach kannst du überall mit `{{storedData.totalProducts}}` auf den Wert zugreifen.

**Persistenter Wert (überlebt Runs):**

```jsonc
{
  "name": "saveLastPage",
  "action": "storeData",
  "params": {
    "key": "lastProcessedPage",
    "value": "{{currentData.pageLoop.index}}",
    "persist": true
  }
}
```

Beim nächsten Run kannst du mit `{{storedData.lastProcessedPage}}` weitermachen, wo du aufgehört hast.

**Mit Handlebars-Template als Wert:**

```jsonc
{
  "name": "saveStatus",
  "action": "storeData",
  "params": {
    "key": "summary",
    "value": "{{previousData.successCount}} von {{previousData.totalCount}} erfolgreich",
    "attachToRun": true
  }
}
```

<Aside type="tip">
Gespeicherte Daten mit `attachToRun: true` tauchen in der Run-Detail-Ansicht im UI auf. Nutze das für eine schnelle Zusammenfassung deines Scrape-Ergebnisses!
</Aside>
