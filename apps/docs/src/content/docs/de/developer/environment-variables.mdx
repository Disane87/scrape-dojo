Hier findest du die vollständige Referenz aller Umgebungsvariablen in Scrape Dojo, sauber nach Kategorie sortiert. Kopier dir raus was du brauchst und pass es an dein Setup an. Am Ende der Seite gibt es eine fertige Production-Config und eine Sicherheits-Checkliste.

## Runtime

| Variable                   | Standard                | Beschreibung                                           |
| -------------------------- | ----------------------- | ------------------------------------------------------ |
| `NODE_ENV`                 | `development`           | Node.js Umgebung (`development`, `production`, `test`) |
| `SCRAPE_DOJO_NODE_ENV`     | `development`           | Scrape Dojo-spezifische Umgebung                       |
| `SCRAPE_DOJO_PORT`         | `3000`                  | API Server Port                                        |
| `SCRAPE_DOJO_DOCKER_ENV`   | `false`                 | Docker-Modus aktivieren                                |
| `SCRAPE_DOJO_TRUST_PROXY`  | `1`                     | Reverse Proxy Trust Level                              |
| `SCRAPE_DOJO_CORS_ORIGIN`  | —                       | CORS Origins (kommasepariert)                          |
| `SCRAPE_DOJO_FRONTEND_URL` | `http://localhost:4200` | Frontend URL für OIDC Redirects                        |
| `EXPORT_OPENAPI`           | `false`                 | OpenAPI Spec beim Start exportieren                    |

## Datenbank

| Variable         | Standard                | Beschreibung                                         |
| ---------------- | ----------------------- | ---------------------------------------------------- |
| `DB_TYPE`        | `sqlite`                | Datenbank-Typ: `sqlite`, `mysql`, `postgres`         |
| `DB_DATABASE`    | `./data/scrape-dojo.db` | Datenbank-Pfad (SQLite) oder Name (MySQL/PostgreSQL) |
| `DB_SYNCHRONIZE` | `true`                  | TypeORM Schema-Sync — erstellt/aktualisiert Tabellen automatisch |
| `DB_LOGGING`     | `false`                 | SQL Query Logging aktivieren                         |
| `DB_HOST`        | —                       | Datenbank-Host (nur MySQL/PostgreSQL)                |
| `DB_PORT`        | `3306`                  | Datenbank-Port (nur MySQL/PostgreSQL)                |
| `DB_USERNAME`    | —                       | Datenbank-Benutzer (nur MySQL/PostgreSQL)            |
| `DB_PASSWORD`    | —                       | Datenbank-Passwort (nur MySQL/PostgreSQL)            |

<Aside type="tip">
  `DB_SYNCHRONIZE=true` ist der Standard und wird beim **ersten Start** benötigt, um alle Tabellen zu erstellen.
  Für laufende Produktiv-Installationen kannst du es auf `false` setzen, sobald die Datenbank initialisiert ist.
  Beachte: Bei Schema-Änderungen (z.B. nach Updates) könnte `synchronize` Spalten droppen und neu erstellen — das kann zu **Datenverlust** führen.
  Langfristig werden Migrations dieses Verhalten ersetzen.
</Aside>

## 🔑 Keys generieren

Für die Verschlüsselungs- und Authentifizierungskeys brauchst du kryptographisch sichere Zufallswerte. So generierst du sie:

```bash
# Encryption Key generieren (256-Bit, 64 Hex-Zeichen)
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# JWT Secret generieren
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# MFA Challenge Secret generieren
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

<Aside type="caution">
  Generiere für jeden Zweck einen **eigenen** Key. Verwende niemals denselben
  Key für Encryption, JWT und MFA!
</Aside>

## Variablen via ENV

Neben der Konfiguration der Anwendung kannst du auch globale Variablen über Umgebungsvariablen definieren. Das ist besonders praktisch für Docker-Deployments.

**Prefix:** `SCRAPE_DOJO_VAR_`

```bash
SCRAPE_DOJO_VAR_DEFAULT_YEAR=2026
SCRAPE_DOJO_VAR_API_ENDPOINT=https://api.example.com
SCRAPE_DOJO_VAR_DEBUG_MODE=true
```

Die Benennung wird automatisch in camelCase konvertiert:

- `SCRAPE_DOJO_VAR_DEFAULT_YEAR` wird zu `defaultYear`
- Zugriff in Scrapes: `{{variables.defaultYear}}`

## Secrets via ENV

Secrets können ebenfalls über Umgebungsvariablen gesetzt werden. Beim Start werden sie automatisch verschlüsselt in der Datenbank gespeichert.

**Prefix:** `SCRAPE_DOJO_SECRET_`

```bash
SCRAPE_DOJO_SECRET_EMAIL=user@example.com
SCRAPE_DOJO_SECRET_PASSWORD=geheimes-passwort
SCRAPE_DOJO_SECRET_API_TOKEN=abc123xyz
```

- `SCRAPE_DOJO_SECRET_EMAIL` wird zu `email`
- Zugriff in Scrapes: `{{secrets.email}}`

## 🏭 Production .env Beispiel

Hier eine minimale `.env` für den Produktivbetrieb:

```bash
# Runtime
NODE_ENV=production
SCRAPE_DOJO_PORT=3000
SCRAPE_DOJO_TRUST_PROXY=1
SCRAPE_DOJO_CORS_ORIGIN=https://scrape.example.com

# Datenbank (PostgreSQL)
DB_TYPE=postgres
DB_HOST=postgres
DB_PORT=5432
DB_USERNAME=scrape_dojo
DB_PASSWORD=<SICHERES_PASSWORT>
DB_DATABASE=scrape_dojo
DB_SYNCHRONIZE=false
DB_LOGGING=false

# Verschlüsselung
SCRAPE_DOJO_ENCRYPTION_KEY=<64_HEX_ZEICHEN>

# Authentifizierung
SCRAPE_DOJO_AUTH_ENABLED=true
SCRAPE_DOJO_AUTH_JWT_SECRET=<MINDESTENS_32_ZEICHEN>
SCRAPE_DOJO_AUTH_REFRESH_TOKEN_SECRET=<MINDESTENS_32_ZEICHEN>
SCRAPE_DOJO_AUTH_ACCESS_TOKEN_EXPIRY=15m
SCRAPE_DOJO_AUTH_REFRESH_TOKEN_EXPIRY=7d

# MFA
SCRAPE_DOJO_AUTH_REQUIRE_MFA=true
SCRAPE_DOJO_AUTH_MFA_CHALLENGE_SECRET=<MINDESTENS_32_ZEICHEN>
SCRAPE_DOJO_AUTH_MFA_ENCRYPTION_KEY=<MINDESTENS_32_ZEICHEN>
```

## ✅ Sicherheits-Checkliste

Bevor du in Production gehst, prüfe diese Punkte:

- [ ] `NODE_ENV=production` gesetzt
- [ ] `DB_SYNCHRONIZE=false` nach der Erstinitialisierung der Datenbank
- [ ] `SCRAPE_DOJO_CORS_ORIGIN` explizit konfiguriert (nicht `*`)
- [ ] Alle Secrets mit `crypto.randomBytes()` generiert
- [ ] PostgreSQL statt SQLite verwenden
- [ ] MFA aktiviert (`SCRAPE_DOJO_AUTH_REQUIRE_MFA=true`)
- [ ] `.env`-Datei in `.gitignore` eingetragen
- [ ] Secrets in einem Passwort-Manager gesichert
- [ ] Rate Limiting konfiguriert
