Du willst Scrape Dojo erweitern? Hier bist du richtig! In diesem Handbuch zeigen wir dir, wie du eigene Actions baust, sie ins System integrierst und vernünftig testest. Das Action-System ist bewusst einfach gehalten — du brauchst nur eine Datei und ein paar Zeilen Code.

## Custom Action erstellen

### Schritt 1: Neue Datei anlegen

Erstelle eine neue Datei im Verzeichnis `apps/api/src/action-handler/actions/`:

```
apps/api/src/action-handler/actions/my-custom.action.ts
```

### Schritt 2: BaseAction erweitern mit @ActionMetadata

Jede Action erbt von `BaseAction` und nutzt den `@ActionMetadata`-Decorator. Der Decorator beschreibt deine Action für die UI und das Factory-System:

| Eigenschaft | Typ | Beschreibung |
|-------------|-----|--------------|
| `name` | String | Technischer Name der Action (z.B. `myCustomAction`) |
| `displayName` | String | Anzeigename in der UI |
| `icon` | String | Icon-Name (Iconify-Format, z.B. `mdi:code-braces`) |
| `color` | String | Farbbezeichnung für die UI |
| `category` | String | Kategorie der Action (siehe Tabelle unten) |
| `description` | String | Kurzbeschreibung der Funktionalität |

### Schritt 3: execute-Methode implementieren

Die zentrale Methode jeder Action ist `execute(page, params, context)`. Sie wird vom ActionHandler aufgerufen und bekommt:

- **page** — Die Puppeteer `Page`-Instanz für Browser-Interaktionen
- **params** — Typisierte Parameter aus der Scrape-Konfiguration
- **context** — Ausführungskontext mit Zugriff auf:
  - `previousData` — Ergebnisse vorheriger Actions
  - `currentData` — Aktuelle Daten (z.B. in Loops)
  - `storedData` — Persistierte Daten aus früheren Runs
  - `variables` — Runtime- und Workflow-Variablen
  - `logger` — ScrapeLogger-Instanz für strukturiertes Logging

### Schritt 4: Automatische Registrierung

Die Action wird automatisch über das Factory Pattern registriert. Du musst nur den Import in `apps/api/src/action-handler/actions/index.ts` hinzufügen — fertig.

## Action-Kategorien

| Kategorie | Beschreibung | Beispiele |
|-----------|--------------|-----------|
| `navigation` | Seitennavigation und URL-Steuerung | navigate, reload, goBack |
| `interaction` | Benutzerinteraktionen simulieren | click, type, select, hover |
| `extraction` | Daten aus Webseiten extrahieren | extract, extractAll, screenshot |
| `flow` | Kontrollfluss und Logik | loop, skipIf, condition |
| `data` | Datenverarbeitung und -transformation | transform, storeData, artifacts |
| `utility` | Hilfsfunktionen | delay, logger, notify, waitForSelector |

## Beispiel: Custom Action Skeleton

```typescript
import { ActionMetadata } from '../_decorators/action.decorator';
import { BaseAction } from './bases/base.action';

interface MyCustomParams {
  selector: string;
  attribute?: string;
  timeout?: number;
}

@ActionMetadata({
  name: 'myCustomAction',
  displayName: 'Meine Custom Action',
  icon: 'mdi:code-braces',
  color: 'blue',
  category: 'extraction',
  description: 'Extrahiert benutzerdefinierte Daten von einer Webseite'
})
export class MyCustomAction extends BaseAction<MyCustomParams> {
  async execute(page, params: MyCustomParams, context): Promise<any> {
    const { selector, attribute = 'textContent', timeout = 30000 } = params;

    // Parameter validieren
    if (!selector) {
      throw new Error('Parameter "selector" ist erforderlich');
    }

    // Auf Element warten
    await page.waitForSelector(selector, { timeout });

    // Daten extrahieren
    const result = await page.$eval(selector, (el, attr) => {
      return el[attr] || el.getAttribute(attr);
    }, attribute);

    // Logging
    context.logger.log(`Extrahiert: ${result}`);

    // Ergebnis zurückgeben (wird in previousData gespeichert)
    return result;
  }
}
```

## Action mit previousData

Actions können auf Ergebnisse vorheriger Actions zugreifen — das ist einer der Kernmechanismen in Scrape Dojo:

```typescript
@ActionMetadata({
  name: 'aggregateData',
  displayName: 'Daten Aggregieren',
  icon: 'mdi:chart-bar',
  color: 'green',
  category: 'data',
  description: 'Aggregiert Daten aus vorherigen Actions'
})
export class AggregateDataAction extends BaseAction<AggregateParams> {
  async execute(page, params: AggregateParams, context): Promise<any> {
    // Zugriff auf Daten vorheriger Actions
    const items = context.previousData.get('extractedItems');

    if (!Array.isArray(items)) {
      throw new Error('Keine Items zum Aggregieren vorhanden');
    }

    return {
      total: items.length,
      sum: items.reduce((acc, item) => acc + (item.value || 0), 0),
      average: items.reduce((acc, item) => acc + (item.value || 0), 0) / items.length
    };
  }
}
```

## Action mit SSE Events

Deine Action kann live Events an die UI schicken — perfekt für Fortschrittsanzeigen:

```typescript
@ActionMetadata({
  name: 'progressAction',
  displayName: 'Fortschritts-Action',
  icon: 'mdi:progress-check',
  color: 'orange',
  category: 'utility',
  description: 'Action mit Fortschrittsanzeige'
})
export class ProgressAction extends BaseAction<ProgressParams> {
  async execute(page, params: ProgressParams, context): Promise<any> {
    const { items } = params;
    const total = items.length;

    for (let i = 0; i < total; i++) {
      // SSE-Event senden
      context.scrapeEventsService?.emitEvent({
        type: 'log',
        scrapeId: context.scrapeId,
        runId: context.runId,
        message: `Verarbeite Element ${i + 1}/${total}`
      });

      await this.processItem(items[i]);
    }

    return { processed: total };
  }

  private async processItem(item: any) {
    // Verarbeitungslogik
  }
}
```

## 🧪 Testing

### Unit Tests mit Vitest

Erstelle eine Testdatei direkt neben der Action:

```typescript
// my-custom.action.spec.ts
import { describe, it, expect, vi } from 'vitest';
import { MyCustomAction } from './my-custom.action';

describe('MyCustomAction', () => {
  it('sollte Daten korrekt extrahieren', async () => {
    const mockPage = {
      waitForSelector: vi.fn().mockResolvedValue(true),
      $eval: vi.fn().mockResolvedValue('Extrahierter Text')
    };

    const mockContext = {
      logger: { log: vi.fn(), debug: vi.fn(), error: vi.fn() },
      previousData: new Map(),
      variables: {}
    };

    const action = new MyCustomAction();
    const result = await action.execute(
      mockPage as any,
      { selector: '.title' },
      mockContext as any
    );

    expect(result).toBe('Extrahierter Text');
    expect(mockPage.waitForSelector).toHaveBeenCalledWith('.title', { timeout: 30000 });
  });

  it('sollte bei fehlendem Selector einen Fehler werfen', async () => {
    const action = new MyCustomAction();

    await expect(
      action.execute({} as any, { selector: '' } as any, {} as any)
    ).rejects.toThrow('Parameter "selector" ist erforderlich');
  });
});
```

## ✅ Best Practices

### Parameter validieren

Prüfe immer die Pflichtparameter am Anfang deiner `execute`-Methode:

```typescript
async execute(page, params, context): Promise<any> {
  const { selector, timeout = 30000 } = params;

  if (!selector) {
    throw new Error('Parameter "selector" ist erforderlich');
  }
  // Weiter mit der Logik
}
```

### Fehlerbehandlung

Fange Fehler ab und gib sinnvolle Meldungen aus:

```typescript
async execute(page, params, context): Promise<any> {
  try {
    const element = await page.$(params.selector);

    if (!element) {
      context.logger.error(`Element nicht gefunden: ${params.selector}`);
      return null;
    }

    return await element.evaluate(el => el.textContent?.trim());
  } catch (error) {
    context.logger.error(`Action fehlgeschlagen: ${error.message}`);
    throw error;
  }
}
```

### TypeScript Types nutzen

Typisiere deine Parameter und Rückgabewerte — das hilft beim Refactoring und macht den Code lesbarer:

```typescript
interface StrictParams {
  required: string;
  optional?: number;
  mode: 'fast' | 'detailed';
}

@ActionMetadata({ name: 'typedAction', /* ... */ })
export class TypedAction extends BaseAction<StrictParams> {
  async execute(page, params: StrictParams, context): Promise<{ success: boolean }> {
    const { required, optional, mode } = params;
    return { success: true };
  }
}
```
