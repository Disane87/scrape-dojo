Du willst direkt loslegen? Perfekt — dieser Guide bringt dich in wenigen Minuten von Null auf dein erstes funktionierendes Scrape. Wir gehen zusammen durch Installation, Konfiguration und den ersten Testlauf.

## 📋 Voraussetzungen

Bevor es losgeht, check kurz ob du folgende Tools installiert hast:

| Software                | Version | Hinweis                                                           |
| ----------------------- | ------- | ----------------------------------------------------------------- |
| **Node.js**             | 20+     | [nodejs.org](https://nodejs.org/)                                 |
| **pnpm**                | 9+      | `npm install -g pnpm`                                             |
| **Chrome/Chromium**     | Aktuell | Wird von Puppeteer für die Browser-Automatisierung genutzt        |
| **Docker** _(optional)_ | 24+     | Empfohlen für Produktion — damit sparst du dir viel Konfig-Arbeit |

<Aside type="tip">
  Wenn du nur mal schnell reinschauen willst, nimm Docker. Damit hast du alles
  in ein paar Minuten am Laufen, ohne dich um Node-Versionen oder Dependencies
  zu kümmern.
</Aside>

## 🚀 Installation

<Tabs>
<TabItem label="🐳 Docker (empfohlen)">

Docker ist der einfachste Weg, Scrape Dojo zum Laufen zu bringen. Kein Node.js nötig, kein Dependency-Chaos — einfach Container starten und fertig.

<Steps>

1. **Encryption Key generieren**

   Der Encryption Key verschlüsselt deine Secrets (Passwörter, API-Keys etc.) in der Datenbank. Du brauchst einen 256-Bit Hex-Key:

   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

   <Aside type="caution">
     Bewahre diesen Key sicher auf (z.B. in einem Passwort-Manager). Wenn du ihn
     verlierst, sind alle gespeicherten Secrets unwiderruflich weg!
   </Aside>

2. **`docker-compose.yml` erstellen**

   Erstelle eine `docker-compose.yml` in einem neuen Ordner:

   Das Image wird in der **GitHub Container Registry (GHCR)** mit Multi-Arch Support (`linux/amd64` + `linux/arm64`) bereitgestellt. Ein einzelner Container enthält API und UI.

   ```yaml
   services:
     scrape-dojo:
       image: ghcr.io/disane87/scrape-dojo:latest
       ports:
         - '8080:80'
       environment:
         - SCRAPE_DOJO_ENCRYPTION_KEY=dein_generierter_key_hier
         - DB_TYPE=sqlite
       volumes:
         - ./data:/home/pptruser/app/data
         - ./downloads:/home/pptruser/app/downloads
         - ./logs:/home/pptruser/app/logs
         - ./config:/home/pptruser/app/config
         - ./browser-data:/home/pptruser/app/browser-data
       restart: unless-stopped
   ```

3. **Container starten**

   ```bash
   docker compose up -d
   ```

4. **Logs checken**

   ```bash
   docker compose logs -f
   ```

   Wenn alles klappt, siehst du nach ein paar Sekunden die Startmeldungen von API und UI.

</Steps>

<Aside type="tip">
  Alles läuft unter `http://localhost:8080` — UI und API (`/api`) auf dem selben
  Port. Einfach im Browser öffnen und loslegen!
</Aside>

**📁 Volumes im Überblick:**

| Volume           | Was steckt drin?                                                      |
| ---------------- | --------------------------------------------------------------------- |
| `./data`         | SQLite-Datenbank und persistente Daten                                |
| `./downloads`    | Heruntergeladene Dateien (PDFs, Screenshots etc.)                     |
| `./logs`         | Log-Dateien der Anwendung                                             |
| `./config`       | Deine Scrape-Konfigurationsdateien (JSONC)                            |
| `./browser-data` | Browser-Profildaten (Cookies, Sessions — damit du eingeloggt bleibst) |

**⚙️ Wichtige Umgebungsvariablen für Docker:**

Im `environment`-Block deiner `docker-compose.yml` kannst du das Verhalten von Scrape Dojo steuern. Hier die wichtigsten:

| Variable | Standard | Beschreibung |
| --- | --- | --- |
| `SCRAPE_DOJO_ENCRYPTION_KEY` | — | **Pflicht.** 256-Bit Hex-Key für die Verschlüsselung von Secrets |
| `DB_TYPE` | `sqlite` | Datenbank-Typ: `sqlite`, `mysql`, `postgres` |
| `DB_SYNCHRONIZE` | `true` | Erstellt Tabellen automatisch beim Start. Nach der Ersteinrichtung auf `false` setzen |
| `SCRAPE_DOJO_AUTH_ENABLED` | `true` | Authentifizierung global aktivieren/deaktivieren |
| `SCRAPE_DOJO_AUTH_JWT_SECRET` | — | JWT-Signing-Secret (mind. 32 Zeichen, in Produktion ändern!) |
| `SCRAPE_DOJO_AUTH_REFRESH_TOKEN_SECRET` | — | Refresh-Token-Secret (mind. 32 Zeichen) |
| `SCRAPE_DOJO_AUTH_REQUIRE_MFA` | `true` | MFA/TOTP für alle Nutzer erzwingen |
| `SCRAPE_DOJO_PORT` | `3000` | Interner API-Port (in Docker normalerweise nicht ändern) |
| `SCRAPE_DOJO_CORS_ORIGIN` | — | CORS-Origins (kommasepariert, für Produktion empfohlen) |
| `SCRAPE_DOJO_AUTH_OIDC_ENABLED` | `false` | OIDC/SSO-Login aktivieren (Keycloak, Auth0, Google, Azure AD) |
| `PUID` | `1000` | User-ID für den Container-Prozess — passend zum Host-User setzen ([Details](/de/getting-started/docker/)) |
| `PGID` | `1000` | Group-ID für den Container-Prozess ([Details](/de/getting-started/docker/)) |
| `UMASK` | `022` | Dateiberechtigungs-Maske für neu erstellte Dateien ([Details](/de/getting-started/docker/)) |

<Aside type="tip">
  Das sind nur die wichtigsten Variablen. Die vollständige Referenz mit allen
  Optionen (Datenbank, Auth, MFA, OIDC, Logging, Rate Limiting) findest du unter
  [Umgebungsvariablen](/de/developer/environment-variables/).
</Aside>

**Beispiel: Produktions-Setup mit Auth und MySQL:**

```yaml
services:
  scrape-dojo:
    image: ghcr.io/disane87/scrape-dojo:latest
    ports:
      - '8080:80'
    environment:
      - SCRAPE_DOJO_ENCRYPTION_KEY=dein_64_zeichen_hex_key
      - DB_TYPE=mysql
      - DB_HOST=db
      - DB_PORT=3306
      - DB_USERNAME=scrape_dojo
      - DB_PASSWORD=sicheres_passwort
      - DB_DATABASE=scrape_dojo
      - SCRAPE_DOJO_AUTH_ENABLED=true
      - SCRAPE_DOJO_AUTH_JWT_SECRET=dein_jwt_secret_mind_32_zeichen
      - SCRAPE_DOJO_AUTH_REFRESH_TOKEN_SECRET=dein_refresh_secret
    volumes:
      - ./downloads:/home/pptruser/app/downloads
      - ./logs:/home/pptruser/app/logs
      - ./config:/home/pptruser/app/config
      - ./browser-data:/home/pptruser/app/browser-data
    restart: unless-stopped
```

</TabItem>
<TabItem label="📦 Aus dem Quellcode">

Wenn du am Code mitarbeiten oder alles selbst kontrollieren willst, kannst du Scrape Dojo direkt aus dem Quellcode starten.

<Steps>

1. **Repository klonen**

   ```bash
   git clone https://github.com/disane87/scrape-dojo.git
   cd scrape-dojo
   ```

2. **Dependencies installieren**

   ```bash
   pnpm install
   ```

   Das dauert beim ersten Mal ein paar Minuten — pnpm lädt alle Pakete für API, UI und die Shared Library.

3. **Environment konfigurieren**

   Erstelle eine `.env`-Datei im Root-Verzeichnis:

   ```bash
   cp .env.example .env
   ```

   Öffne die `.env` und setze mindestens den Encryption Key:

   ```bash
   SCRAPE_DOJO_ENCRYPTION_KEY=dein_generierter_key_hier
   ```

   Key generieren:

   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

4. **Anwendung starten**

   ```bash
   # Alles auf einmal (API + UI)
   pnpm start

   # Oder einzeln, wenn du nur eins brauchst:
   pnpm start:api   # API auf Port 3000
   pnpm start:ui    # UI auf Port 4200
   ```

</Steps>

<Aside type="tip">
  Im Dev-Modus läuft die UI unter `http://localhost:4200`. Der Dev-Server leitet
  API-Anfragen automatisch an `http://localhost:3000` weiter — du musst dich
  also nicht um CORS oder Proxy-Konfig kümmern.
</Aside>

</TabItem>
</Tabs>

## ⚡ Dein erstes Scrape

Jetzt wird's spannend! Stell dir vor, du willst den Titel einer Webseite auslesen. Erstelle dafür eine neue Datei `config/sites/my-first-scrape.jsonc`:

```jsonc
{
  "$schema": "../scrapes.schema.json",
  "scrapes": [
    {
      "id": "my-first-scrape",
      "metadata": {
        "description": "Mein erstes Scrape — liest den Seitentitel aus",
        "version": "1.0.0",
        "author": "gh:@dein-username",
        "triggers": [{ "type": "manual" }],
      },
      "steps": [
        {
          "name": "Titel auslesen",
          "actions": [
            {
              "name": "navigate",
              "action": "navigate",
              "params": { "url": "https://example.com" },
            },
            {
              "name": "title",
              "action": "extract",
              "params": { "selector": "h1" },
            },
            {
              "name": "logTitle",
              "action": "logger",
              "params": {
                "message": "Gefundener Titel: {{previousData.title}}",
                "level": "log",
              },
            },
          ],
        },
      ],
    },
  ],
}
```

### 🔍 Was passiert hier?

Das Scrape führt drei Actions nacheinander aus:

| #   | Action     | Was sie macht                                                                                                                                       |
| --- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | `navigate` | Öffnet `https://example.com` im Browser                                                                                                             |
| 2   | `extract`  | Holt den Text aus dem `h1`-Element. Das Ergebnis landet unter `previousData.title` (der `name` der Action ist der Key)                              |
| 3   | `logger`   | Gibt den Titel in der Konsole aus. Das `{{previousData.title}}` ist ein Handlebars-Template — damit greifst du auf Ergebnisse vorheriger Actions zu |

<Aside type="note">
  Der `name` jeder Action ist super wichtig: Er bestimmt, unter welchem Key das
  Ergebnis gespeichert wird. Wähle also aussagekräftige Namen wie `title`,
  `productLinks` oder `priceData`.
</Aside>

## ▶️ Scrape starten

### 🖥️ Über die UI

<Steps>

1. Öffne die UI im Browser:
   - **Docker**: `http://localhost:8080`
   - **Entwicklung**: `http://localhost:4200`

2. Du siehst dein Scrape "my-first-scrape" in der Übersicht — Scrape Dojo erkennt neue Config-Dateien automatisch (Hot Reload).

3. Klicke auf das Scrape, um die Detailansicht zu öffnen.

4. Klicke auf **"Run"**, um das Scrape zu starten.

5. Beobachte die Live-Ausführung in Echtzeit über Server-Sent Events (SSE) direkt in der UI. Du siehst jeden Step und jede Action mit Status und Ergebnis.

</Steps>

### 🌐 Über die API

Du kannst Scrapes auch direkt per REST-API starten — praktisch für Automatisierung oder Debugging:

```bash
# Scrape starten
curl http://localhost:3000/api/scrape/my-first-scrape
```

<Aside type="note">
  Im Docker-Setup erreichst du die API über nginx auf Port 8080:
  `http://localhost:8080/api/scrape/my-first-scrape`
</Aside>

Du kannst beim Start auch Variablen mitgeben:

```bash
curl -X POST http://localhost:3000/api/scrape/my-first-scrape \
  -H "Content-Type: application/json" \
  -d '{ "variables": { "targetUrl": "https://example.org" } }'
```
