# Authentication Flow

Scrape Dojo unterstützt mehrere Authentifizierungsmethoden: lokale JWT-basierte Authentifizierung, OpenID Connect (OIDC/SSO) und Multi-Faktor-Authentifizierung (MFA/TOTP).

## Lokale Authentifizierung (JWT)

### Login-Flow

```mermaid
sequenceDiagram
    autonumber
    participant U as 👤 User
    participant UI as 🎨 UI
    participant API as ⚙️ API
    participant Auth as 🔐 AuthService
    participant DB as 💾 Database

    U->>UI: Eingabe: Username & Password
    UI->>API: POST /api/auth/login
    API->>Auth: validateUser(username, password)
    Auth->>DB: findUserByUsername(username)
    DB-->>Auth: User entity
    
    Auth->>Auth: bcrypt.compare(password, hash)
    
    alt Password korrekt
        Auth-->>API: User validated ✅
        API->>Auth: generateJWT(user)
        Auth->>Auth: Sign JWT Token<br/>(Secret: JWT_SECRET)
        Auth-->>API: JWT Token
        API-->>UI: { access_token, user }
        UI->>UI: Store Token (Cookie/LocalStorage)
        UI-->>U: Redirect zu Dashboard
    else Password falsch
        Auth-->>API: Unauthorized ❌
        API-->>UI: 401 Error
        UI-->>U: Anzeige Fehler
    end
```

### Request Authentication

```mermaid

    autonumber
    participant C as Client
    participant API as API
    participant Guard as JwtAuthGuard
    participant Strategy as JwtStrategy
    participant DB as Database

    C->>API: Request + Authorization Header
    Note over C,API: Bearer eyJhbGc...
    
    API->>Guard: canActivate()
    Guard->>Strategy: validate(payload)
    Strategy->>Strategy: Verify JWT Signature
    
    alt Token valid
        Strategy->>DB: findUserById(payload.sub)
        DB-->>Strategy: User
        Strategy-->>Guard: User object
        Guard-->>API: true ✅

        API-->>C: Response
    else Token invalid/expired
        Strategy-->>Guard: Error
        Guard-->>API: false ❌
        API-->>C: 401 Unauthorized
    end
```

## OIDC/SSO Authentifizierung

### OIDC Authorization Code Flow

```mermaid
sequenceDiagram
    autonumber
    participant U as 👤 User
    participant UI as 🎨 Scrape Dojo UI
    participant API as ⚙️ Scrape Dojo API
    participant OIDC as 🌐 OIDC Provider

    U->>UI: Click "Login with SSO"
    UI->>API: GET /api/auth/oidc
    API->>API: Generate state & nonce
    API-->>UI: Redirect URL
    UI-->>U: Redirect to OIDC Provider
    
    U->>OIDC: Login at Provider
    OIDC->>OIDC: User authenticates
    OIDC-->>U: Redirect with auth code
    
    U->>API: GET /api/auth/oidc/callback?code=...
    API->>OIDC: POST /token<br/>(exchange code for tokens)
    OIDC-->>API: { access_token, id_token }
    
    API->>OIDC: GET /userinfo
    OIDC-->>API: User profile
    
    API->>API: Find or create user
    API->>API: Generate local JWT
    API-->>UI: Set JWT cookie
    UI-->>U: Redirect to Dashboard
```

## Multi-Faktor Authentifizierung (MFA/TOTP)

### MFA Setup Flow

```mermaid
sequenceDiagram
    autonumber
    participant U as 👤 User
    participant UI as 🎨 UI
    participant API as ⚙️ API
    participant TOTP as 🔐 TOTP Service

    U->>UI: Enable MFA
    UI->>API: POST /api/users/totp/setup
    API->>TOTP: generateSecret()
    TOTP-->>API: { secret, qrCode }
    API-->>UI: { secret, qrCodeDataUrl }
    
    UI->>U: Display QR Code
    U->>U: Scan with Authenticator App
    
    U->>UI: Enter verification code
    UI->>API: POST /api/users/totp/verify<br/>{ code }
    
    API->>TOTP: verifyToken(secret, code)
    
    alt Code valid
        TOTP-->>API: true ✅
        API->>API: Save encrypted secret to DB
        API-->>UI: MFA enabled
        UI-->>U: Show backup codes
    else Code invalid
        TOTP-->>API: false ❌
        API-->>UI: 400 Error
        UI-->>U: Invalid code
    end
```

### MFA Login Flow

```mermaid
sequenceDiagram
    autonumber
    participant U as 👤 User
    participant UI as 🎨 UI
    participant API as ⚙️ API
    participant Auth as 🔐 AuthService
    participant TOTP as 🔢 TOTP Service
    participant DB as 💾 Database

    U->>UI: Username & Password
    UI->>API: POST /api/auth/login
    API->>Auth: validateUser(credentials)
    Auth->>DB: Find user
    DB-->>Auth: User (with totpSecret)
    
    Auth->>Auth: Verify password
    
    alt MFA enabled
        Auth-->>API: MFA required
        API-->>UI: { requiresMfa: true }
        
        UI->>U: Request TOTP code
        U->>UI: Enter 6-digit code
        UI->>API: POST /api/auth/mfa/verify<br/>{ userId, code }
        
        API->>TOTP: verifyToken(totpSecret, code)
        
        alt TOTP valid
            TOTP-->>API: true ✅
            API->>API: Generate JWT
            API-->>UI: { access_token }
            UI-->>U: Login successful
        else TOTP invalid
            TOTP-->>API: false ❌
            API-->>UI: 401 Error
            UI-->>U: Invalid code
        end
        
    else No MFA
        Auth->>API: Generate JWT
        API-->>UI: { access_token }
        UI-->>U: Login successful
    end
```

### TOTP Algorithm

```mermaid

    Truncate --> Code["🔢 6-Digit Code<br/>(modulo 1,000,000)"]
    
    Code --> Validate{Code<br/>matches?}
```

## Best Practices

### ✅ Do's
- Verwende starke JWT Secrets (mindestens 32 Zeichen)
- Aktiviere MFA für erhöhte Sicherheit
- Rotiere TOTP Secrets bei Verdacht auf Kompromittierung
- Nutze HTTPS in Produktion
- Setze angemessene Token-Ablaufzeiten

### ❌ Don'ts
- Speichere Passwörter im Klartext
- Teile JWT Secrets in Code-Repositories
- Verwende schwache Passwörter
- Deaktiviere CORS-Protection
- Logge sensitive Daten
