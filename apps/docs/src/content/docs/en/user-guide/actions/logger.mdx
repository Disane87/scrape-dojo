Debugging without logs is like driving with your eyes closed. The `logger` action writes messages to the log that you can see in the run history and in real time in the UI. Use it generously -- it helps enormously with troubleshooting.

Messages support Handlebars templates, so you can embed dynamic values directly.

## Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message` | `string` | Yes | The log message. Supports Handlebars templates. |
| `level` | `string` | No | Log level: `log`, `error`, `warn`, `debug`, `verbose` (default: `log`) |

### Log Levels

| Level | Description |
|-------|-------------|
| `log` | Normal informational message (default) |
| `error` | Error message -- highlighted in red |
| `warn` | Warning -- highlighted in yellow |
| `debug` | Debug info -- only visible when debug mode is enabled |
| `verbose` | Detailed info -- only visible at the highest log level |

## Examples

**Simple status message:**

```jsonc
{
  "name": "logStart",
  "action": "logger",
  "params": {
    "message": "Scrape started, here we go!",
    "level": "log"
  }
}
```

**Error message:**

```jsonc
{
  "name": "logError",
  "action": "logger",
  "params": {
    "message": "Could not find element -- page has probably changed",
    "level": "error"
  }
}
```

**Debug with previousData:**

```jsonc
{
  "name": "logDebug",
  "action": "logger",
  "params": {
    "message": "Extracted data: {{previousData.extractedItems}}",
    "level": "debug"
  }
}
```

**Progress in a loop:**

```jsonc
{
  "name": "logProgress",
  "action": "logger",
  "params": {
    "message": "Processing page {{add currentData.pageLoop.index 1}} of {{variables.maxPages}}",
    "level": "log"
  }
}
```

<Aside type="tip">
Place logger actions at important points in the workflow -- e.g. after an extraction, before a download, or at the end of a loop. In the run history, you'll then see exactly what happened.
</Aside>

## Tips

- The `logger` action does not return a value (i.e. `undefined`).
- Log messages appear both in the run history and in real-time SSE events.
- Use `debug` and `verbose` for detailed information that you only want to see when needed.
- You can also log computed values using the `jsonata` Handlebars helper:
  ```jsonc
  {
    "name": "logCount",
    "action": "logger",
    "params": {
      "message": "Found: {{jsonata previousData.items '$count($)'}} entries",
      "level": "log"
    }
  }
  ```
