The `type` action lets you enter text into input fields тАФ just like you would manually via the keyboard. Ideal for login forms, search fields, or any other text input. Bonus: you can include secrets and even define conditions for when typing should happen.

## тЪЩя╕П Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `selector` | `string` | Yes | тАФ | CSS selector of the input field. |
| `text` | `string` | Yes | тАФ | The text to enter. Supports Handlebars templates. |
| `pressEnter` | `boolean` | No | `false` | If `true`, Enter is pressed after typing (triggers navigation). |
| `condition` | `string` | No | тАФ | Handlebars condition. The action only runs if the result is `"true"`. |

## ЁЯТб Examples

### ЁЯОп Simple Text Input

```jsonc
{
  "name": "username",
  "action": "type",
  "params": {
    "selector": "input[name='username']",
    "text": "john.doe@example.com"
  }
}
```

### ЁЯФТ Password from Secrets

You should never write sensitive data like passwords directly into the config. Use secrets instead:

```jsonc
{
  "name": "password",
  "action": "type",
  "params": {
    "selector": "input[type='password']",
    "text": "{{secrets.portalPassword}}"
  }
}
```

<Aside type="caution">
Password fields are automatically masked in the log (`********`) when the selector contains the word "password". Still тАФ always use secrets instead of plaintext!
</Aside>

### тПО Input with Enter

Handy for search fields that don't have their own submit button:

```jsonc
{
  "name": "search",
  "action": "type",
  "params": {
    "selector": "input[name='search']",
    "text": "Scrape Dojo",
    "pressEnter": true
  }
}
```

### ЁЯФА Conditional Input

Sometimes you only want to type when a certain condition is met тАФ for example, when a previous field has a specific value:

```jsonc
{
  "name": "optionalField",
  "action": "type",
  "params": {
    "selector": "input[name='extra']",
    "text": "{{variables.extraValue}}",
    "condition": "{{isDefined previousData.emailField}}"
  }
}
```

The `condition` is evaluated as a Handlebars template. The input only runs if the result is the string `"true"`. Otherwise, the action is skipped.

<Aside type="tip">
The `type` action automatically clears the input field before typing. So you don't need to worry about old text sticking around. If direct typing doesn't work (some React apps can be picky), Scrape Dojo automatically falls back to a JavaScript-based approach.
</Aside>
