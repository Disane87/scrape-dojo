Want to jump right in? Perfect — this guide takes you from zero to your first working scrape in just a few minutes. We'll walk through installation, configuration, and your first test run together.

## 📋 Prerequisites

Before we get started, quickly check that you have the following tools installed:

| Software                | Version | Note                                                          |
| ----------------------- | ------- | ------------------------------------------------------------- |
| **Node.js**             | 20+     | [nodejs.org](https://nodejs.org/)                             |
| **pnpm**                | 9+      | `npm install -g pnpm`                                         |
| **Chrome/Chromium**     | Latest  | Used by Puppeteer for browser automation                      |
| **Docker** _(optional)_ | 24+     | Recommended for production — saves you a lot of config hassle |

<Aside type="tip">
  If you just want to take a quick look, go with Docker. You'll have everything
  up and running in a few minutes without worrying about Node versions or
  dependencies.
</Aside>

## 🚀 Installation

<Tabs>
<TabItem label="🐳 Docker (recommended)">

Docker is the easiest way to get Scrape Dojo up and running. No Node.js needed, no dependency chaos — just start the containers and you're good to go.

<Steps>

1. **Generate an encryption key**

   The encryption key encrypts your secrets (passwords, API keys, etc.) in the database. You need a 256-bit hex key:

   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

   <Aside type="caution">
     Store this key safely (e.g., in a password manager). If you lose it, all
     stored secrets are gone for good!
   </Aside>

2. **Create a `docker-compose.yml`**

   Create a `docker-compose.yml` in a new folder:

   The image is published to **GitHub Container Registry (GHCR)** with multi-arch support (`linux/amd64` + `linux/arm64`). A single container runs both API and UI.

   ```yaml
   services:
     scrape-dojo:
       image: ghcr.io/disane87/scrape-dojo:latest
       ports:
         - '8080:80'
       environment:
         - SCRAPE_DOJO_ENCRYPTION_KEY=your_generated_key_here
         - DB_TYPE=sqlite
       volumes:
         - ./data:/home/pptruser/app/data
         - ./downloads:/home/pptruser/app/downloads
         - ./logs:/home/pptruser/app/logs
         - ./config:/home/pptruser/app/config
         - ./browser-data:/home/pptruser/app/browser-data
       restart: unless-stopped
   ```

3. **Start the containers**

   ```bash
   docker compose up -d
   ```

4. **Check the logs**

   ```bash
   docker compose logs -f
   ```

   If everything works, you'll see the startup messages from the API and UI after a few seconds.

</Steps>

<Aside type="tip">
  Everything runs at `http://localhost:8080` — UI and API (`/api`) on the same
  port. Just open it in your browser and get going!
</Aside>

**📁 Volumes overview:**

| Volume           | What's inside?                                                   |
| ---------------- | ---------------------------------------------------------------- |
| `./data`         | SQLite database and persistent data                              |
| `./downloads`    | Downloaded files (PDFs, screenshots, etc.)                       |
| `./logs`         | Application log files                                            |
| `./config`       | Your scrape configuration files (JSONC)                          |
| `./browser-data` | Browser profile data (cookies, sessions — so you stay logged in) |

**⚙️ Important environment variables for Docker:**

Use the `environment` block in your `docker-compose.yml` to configure Scrape Dojo's behavior. Here are the most important ones:

| Variable | Default | Description |
| --- | --- | --- |
| `SCRAPE_DOJO_ENCRYPTION_KEY` | — | **Required.** 256-bit hex key for encrypting secrets |
| `DB_TYPE` | `sqlite` | Database type: `sqlite`, `mysql`, `postgres` |
| `SCRAPE_DOJO_AUTH_ENABLED` | `true` | Enable/disable authentication globally |
| `SCRAPE_DOJO_AUTH_JWT_SECRET` | — | JWT signing secret (min. 32 chars, change in production!) |
| `SCRAPE_DOJO_AUTH_REFRESH_TOKEN_SECRET` | — | Refresh token secret (min. 32 chars) |
| `SCRAPE_DOJO_AUTH_REQUIRE_MFA` | `true` | Require MFA/TOTP for all users |
| `SCRAPE_DOJO_PORT` | `3000` | Internal API port (usually no need to change in Docker) |
| `SCRAPE_DOJO_CORS_ORIGIN` | — | CORS origins (comma-separated, recommended for production) |
| `SCRAPE_DOJO_AUTH_OIDC_ENABLED` | `false` | Enable OIDC/SSO login (Keycloak, Auth0, Google, Azure AD) |

<Aside type="tip">
  These are just the most important variables. For the full reference with all
  options (database, auth, MFA, OIDC, logging, rate limiting), see
  [Environment Variables](/en/developer/environment-variables/).
</Aside>

**Example: Production setup with auth and MySQL:**

```yaml
services:
  scrape-dojo:
    image: ghcr.io/disane87/scrape-dojo:latest
    ports:
      - '8080:80'
    environment:
      - SCRAPE_DOJO_ENCRYPTION_KEY=your_64_char_hex_key
      - DB_TYPE=mysql
      - DB_HOST=db
      - DB_PORT=3306
      - DB_USERNAME=scrape_dojo
      - DB_PASSWORD=secure_password
      - DB_DATABASE=scrape_dojo
      - SCRAPE_DOJO_AUTH_ENABLED=true
      - SCRAPE_DOJO_AUTH_JWT_SECRET=your_jwt_secret_min_32_chars
      - SCRAPE_DOJO_AUTH_REFRESH_TOKEN_SECRET=your_refresh_secret
    volumes:
      - ./downloads:/home/pptruser/app/downloads
      - ./logs:/home/pptruser/app/logs
      - ./config:/home/pptruser/app/config
      - ./browser-data:/home/pptruser/app/browser-data
    restart: unless-stopped
```

</TabItem>
<TabItem label="📦 From source">

If you want to contribute to the code or have full control over everything, you can start Scrape Dojo directly from the source.

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/disane87/scrape-dojo.git
   cd scrape-dojo
   ```

2. **Install dependencies**

   ```bash
   pnpm install
   ```

   This takes a few minutes the first time — pnpm downloads all packages for the API, UI, and the shared library.

3. **Configure the environment**

   Create a `.env` file in the root directory:

   ```bash
   cp .env.example .env
   ```

   Open the `.env` and set at least the encryption key:

   ```bash
   SCRAPE_DOJO_ENCRYPTION_KEY=your_generated_key_here
   ```

   Generate the key:

   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

4. **Start the application**

   ```bash
   # Everything at once (API + UI)
   pnpm start

   # Or individually, if you only need one:
   pnpm start:api   # API on port 3000
   pnpm start:ui    # UI on port 4200
   ```

</Steps>

<Aside type="tip">
  In dev mode, the UI runs at `http://localhost:4200`. The dev server
  automatically proxies API requests to `http://localhost:3000` — so you don't
  need to worry about CORS or proxy config.
</Aside>

</TabItem>
</Tabs>

## ⚡ Your first scrape

Now it gets exciting! Imagine you want to grab the title of a webpage. Create a new file `config/sites/my-first-scrape.jsonc`:

```jsonc
{
  "$schema": "../scrapes.schema.json",
  "scrapes": [
    {
      "id": "my-first-scrape",
      "metadata": {
        "description": "My first scrape — reads the page title",
        "version": "1.0.0",
        "author": "gh:@your-username",
        "triggers": [{ "type": "manual" }],
      },
      "steps": [
        {
          "name": "Read title",
          "actions": [
            {
              "name": "navigate",
              "action": "navigate",
              "params": { "url": "https://example.com" },
            },
            {
              "name": "title",
              "action": "extract",
              "params": { "selector": "h1" },
            },
            {
              "name": "logTitle",
              "action": "logger",
              "params": {
                "message": "Found title: {{previousData.title}}",
                "level": "log",
              },
            },
          ],
        },
      ],
    },
  ],
}
```

### 🔍 What's happening here?

The scrape runs three actions in sequence:

| #   | Action     | What it does                                                                                                                                  |
| --- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | `navigate` | Opens `https://example.com` in the browser                                                                                                    |
| 2   | `extract`  | Grabs the text from the `h1` element. The result is stored under `previousData.title` (the action's `name` is the key)                        |
| 3   | `logger`   | Outputs the title to the console. The `{{previousData.title}}` is a Handlebars template — that's how you access results from previous actions |

<Aside type="note">
  The `name` of each action is super important: it determines which key the
  result is stored under. So pick descriptive names like `title`,
  `productLinks`, or `priceData`.
</Aside>

## ▶️ Start the scrape

### 🖥️ Via the UI

<Steps>

1. Open the UI in your browser:
   - **Docker**: `http://localhost:8080`
   - **Development**: `http://localhost:4200`

2. You'll see your scrape "my-first-scrape" in the overview — Scrape Dojo automatically picks up new config files (hot reload).

3. Click on the scrape to open the detail view.

4. Click **"Run"** to start the scrape.

5. Watch the live execution in real-time via Server-Sent Events (SSE) right in the UI. You'll see every step and every action with its status and result.

</Steps>

### 🌐 Via the API

You can also start scrapes directly via the REST API — handy for automation or debugging:

```bash
# Start the scrape
curl http://localhost:3000/api/scrape/my-first-scrape
```

<Aside type="note">
  In the Docker setup, the API is accessible via nginx on port 8080:
  `http://localhost:8080/api/scrape/my-first-scrape`
</Aside>

You can also pass variables when starting:

```bash
curl -X POST http://localhost:3000/api/scrape/my-first-scrape \
  -H "Content-Type: application/json" \
  -d '{ "variables": { "targetUrl": "https://example.org" } }'
```
