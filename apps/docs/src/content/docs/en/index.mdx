## How It Works

Instead of writing hundreds of lines of Puppeteer code, you describe your workflow declaratively:

```jsonc
{
  "scrapes": [{
    "id": "invoice-download",
    "metadata": {
      "description": "Download invoices automatically",
      "triggers": [{ "type": "cron" }]
    },
    "steps": [{
      "name": "Login and Download",
      "actions": [
        { "name": "open", "action": "navigate",
          "params": { "url": "https://portal.example.com/login" } },
        { "name": "email", "action": "type",
          "params": { "selector": "#email", "text": "{{variables.email}}", "pressEnter": false } },
        { "name": "password", "action": "type",
          "params": { "selector": "#password", "text": "{{variables.password}}", "pressEnter": true } },
        { "name": "invoices", "action": "extractAll",
          "params": { "selector": ".invoice-link", "property": "href" } },
        { "name": "downloadLoop", "action": "loop",
          "params": {
            "elementKey": "invoices",
            "actions": [
              { "name": "dl", "action": "download",
                "params": {
                  "url": "{{currentData.downloadLoop.value}}",
                  "path": "./invoices",
                  "filename": "invoice-{{currentData.downloadLoop.index}}.pdf"
                }}
            ]
          }}
      ]
    }]
  }]
}
```

Secrets are AES-256-CBC encrypted, the UI shows progress live via SSE, and cron scheduling runs jobs automatically.

## Core Features

<CardGrid>
  <Card title="JSONC Configuration" icon="seti:json">
    Workflows as JSON with comments. Schema validation, hot reload, no code required.
  </Card>
  <Card title="25+ Actions" icon="puzzle">
    Navigate, Click, Type, Extract, Loop, Transform, Download, Screenshot, Notify and more.
  </Card>
  <Card title="Templates and Transforms" icon="setting">
    Handlebars for string interpolation, JSONata for data transformations, 13 custom helpers.
  </Card>
  <Card title="Cron Scheduling" icon="rocket">
    Automatic execution via cron with timezone support. Plus webhook and startup triggers.
  </Card>
  <Card title="Real-time Monitoring" icon="laptop">
    Angular 21 UI with live status via SSE, run history, debug data and artifact display.
  </Card>
  <Card title="Security" icon="shield">
    AES-256-CBC secrets, JWT auth, OIDC/SSO, MFA/TOTP, API keys. Auth optionally disableable.
  </Card>
  <Card title="Multi-Database" icon="document">
    SQLite (default), PostgreSQL or MySQL via TypeORM. Docker-ready with persistent volumes.
  </Card>
  <Card title="Extensible" icon="add-document">
    Create custom actions in TypeScript. Factory pattern, auto-registration, full Puppeteer access.
  </Card>
</CardGrid>

<Aside type="caution" title="Responsible Use">
Scrape Dojo automates real browser interactions. Please respect website Terms of Service, robots.txt, rate limits and privacy laws (GDPR).
</Aside>

<Aside type="note" title="AI-Aided Development (AIAD)">
This project openly uses AI-assisted development (e.g. [Claude Code](https://claude.ai/code)) to accelerate workflows, improve code quality, and gain more development momentum. All AI-generated code is **reviewed and approved by humans** — this is not a vibe-coding project, but a deliberate effort to build a useful product while exploring the boundaries, benefits, and trade-offs of AI-aided development.
</Aside>
